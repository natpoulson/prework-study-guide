HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Semantic HTML refers to elements that describe the context of their contents. This is useful for assistive technologies such as screen readers, as it helps organise the order in which content is presented or narrated.
  • The h1 to h6 tags are used to mark headings for content. As a rule-of-thumb, the h1 tag should only be used once.
  • All HTML elements can have attributes, which are included in the opening section of a tag. Examples of attributes include "class", "id", and "href".
  • The position of the opening and closing tags (on elements that use them) are used to indicate the hierarchy of elements. Elements placed in the space between another element's tags are considered to be children of that element. An example of such a relationship is the "body" tag, which contains the majority of a page's content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • Using the auto value for margin will cause it to automatically take up available space on the side(s) it was specified for, provided a fixed height and width are defined.
  • padding indicates how much space we want around the content inside an element.
  • The representation of the components within an element is commonly referred to as the "box model".
  • From outermost to innermost, the box model comprises of: margins, borders, padding, and content.
  • The box-sizing property dictates how the sizes of elements are calculated in the browser, and they come in two different modes:
    • content-box (Default) - Calculates element size based on the content only, both padding and margins are calculated independently meaning the box itself can be much larger than its defined width and height.
    • border-box - Calculates element size up to the border of the box. The padding shares space with the content with its value representing a fixed or relative ratio of space between the content and the border, and the margin defines the space between the element and others.
  • The asterisk * is a universal selector and applies to all elements.
  • Multiple elements can be selected in one rule by delimiting each one with a comma. This can be used in conjunction with elements that have ID, Class, and other selectors.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add <file or folder>: marks a file for staging
  • git commit -m "<comment>": capture all staged files as a snapshot
  • git push <remote alias> <branch>: upload local commits to remote repository
  • git pull --all: download the latest version of all remote repositories and branches

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script. Examples of control flows include:
    • Conditionals - Such as if and switch...case.
    • Loops - Such as for, while, and do...until.
  • Variable types:
    • var - The original, globally-scoped variable type. This means that any part of the code can access or modify the value contained. Not recommended in most cases for modern JS.
    • let - Mutable, variable type, which allows for its data to be changed. Used when you need to be able to change the data type or contents of certain types like Strings and Numbers.
    • const - (Mostly) immutable variable. Once created, the value (excepting in the case of the contents of arrays and objects) cannot be modified.
  • As a rule of thumb, you should use const as often as possible, use let for variables that need to be able to change, and var only in situations where neither of the aformentioned are suitable.
  • Scope defines the range with which a variable can be accessed. There are three scopes - Global, Function, and Block
    • Global - Variable can be accessed from anywhere within the script or file it's declared in. For example:
      • Any variables declared anywhere outside any function or code block.
      • Any variable declared with var.
    • Function - Variable can be accessed from anywhere within a function, or blocks within the function. For example:
      • A function called toast() with the variable temperature, can be used by any part of the code inside toast() but not in another function called addBread().
    • Block - Variable can only be accessed from within the block of code it's running from, such as an if statement, or for loop. For example:
      • A variable in a for loop called i, which is used to track the steps, can't be used by the function it's contained in.
  • Variables in a given scope can be accessed by functions or code blocks nested under them in the hierarchy (Global > Function > Block). These capabilities are referred to as Closures.
    • The process of accessing a value in Closure is known as Hoisting.
  • Unlike CSS, which should be declared as early in the document flow as possible (in the head), script tags should be declared in the body at the end. This allows the DOM time to load in so code such as event listeners can be registered successfully.
